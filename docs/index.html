Quick Start

Get the MCP server running in under two minutes. You need Node.js 18+ and a Prospeo API key.

Claude Desktop

Open ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) or %APPDATA%\Claude\claude_desktop_config.json (Windows) and add:

json — claude_desktop_config.json 
{
  "mcpServers": {
    "prospeo": {
      "command": "npx",
      "args": ["-y", "@prospeo/mcp-server"],
      "env": {
        "PROSPEO_API_KEY": "your_api_key_here"
      }
    }
  }
}

Restart Claude Desktop. The Prospeo tools will appear in the tools menu.

Claude Code (CLI)

bash 
# Add the MCP server with your API key
claude mcp add prospeo \
  --env PROSPEO_API_KEY=your_api_key_here \
  -- npx -y @prospeo/mcp-server

Cursor / VS Code

Add to your .cursor/mcp.json or VS Code MCP settings:

json — .cursor/mcp.json 
{
  "mcpServers": {
    "prospeo": {
      "command": "npx",
      "args": ["-y", "@prospeo/mcp-server"],
      "env": {
        "PROSPEO_API_KEY": "your_api_key_here"
      }
    }
  }
}
✅

Verify the server is working by asking your AI: "Check my Prospeo account credits". It should call get_account_info and display your plan details.

Authentication

All requests to the Prospeo API are authenticated with your API key, passed as the X-KEY HTTP header. The MCP server handles this automatically — you only need to provide the key as an environment variable.

1

Get your API key

Log in to app.prospeo.io → API Management and generate or copy your key.

2

Set PROSPEO_API_KEY

Pass the key as an environment variable when registering the MCP server (see Quick Start). The variable name must be exactly PROSPEO_API_KEY.

3

Key is validated at startup

The server fails immediately with a clear message if PROSPEO_API_KEY is missing or empty — so you'll know right away if something is wrong.

⚠️

Never commit your API key to source control. Use environment variable injection via your MCP client's config — not hardcoded strings.

Configuration

All options are set via environment variables. Only PROSPEO_API_KEY is required.

Variable Default Description
PROSPEO_API_KEY required Your Prospeo API key
PROSPEO_API_BASE_URL https://api.prospeo.io Base URL for the Prospeo API. Override for testing against staging.
PROSPEO_TIMEOUT 30000 HTTP request timeout in milliseconds.
LOG_LEVEL INFO Log verbosity: DEBUG, INFO, WARN, or ERROR. Logs go to stderr.

1 or 10 credits

Provide at least one of: linkedin_url, email, or (first_name/full_name + company_name/company_website). Requests with no identifying data are rejected.

Filter Reference

Both search_person and search_company accept filters as a plain JSON object. Filter keys use one of the shapes below depending on the field.

Filter shapes
ShapeStructureDescription
include/exclude { include?: string[], exclude?: string[] } Results must match at least one include value; any exclude match removes the result.
string[] string[] A plain array of accepted string values (e.g. headcount brackets).
range { min?: number, max?: number } Numeric lower/upper bound. Both fields are optional.
string string A single string value (e.g. company_type, person_name_or_job_title).
object varies per key Richer structure — see the per-key descriptions in the tool sections above.
json — composite example 
{
  "filters": {
    "person_job_title": {
      "include": ["CTO", "VP of Engineering"]
    },
    "person_seniority": {
      "include": ["C-Suite", "VP"],
      "exclude": ["Intern"]
    },
    "person_location_search": {
      "include": ["United States"],
      "exclude": ["Texas, United States"]
    },
    "company_headcount_range": ["51-100", "101-200", "201-500"],
    "company_industry": {
      "include": ["Computer Software", "Information Technology and Services"]
    },
    "company_revenue": { "min": "10M", "max": "500M" },
    "company_technology": {
      "include": ["Salesforce"]
    }
  },
  "page": 1
}
⚠️

An empty filters object ({}) will be rejected with INVALID_REQUEST. At least one filter key must be provided.

Error Codes

When a tool call fails, the server returns a structured error object:

json — error response shape 
{
  "success": false,
  "error": {
    "message":  "Human-readable explanation",
    "code":     "INSUFFICIENT_CREDITS",
    "category": "api_client_error",
    "severity": "high"
  }
}
Code Category Severity Meaning & fix
INSUFFICIENT_CREDITS api_client_error high Account has no credits left. Upgrade your plan or wait for the next billing cycle.
INVALID_DATAPOINTS validation low No identifying data provided for enrichment. Add at least one of linkedin_url, email, or name + company.
INVALID_REQUEST validation low Malformed request body (e.g. empty filters, out-of-range page number).
INVALID_FILTERS validation low One or more filter keys or values are not recognized. Check the Filter Reference reference.
INVALID_API_KEY configuration critical API key is invalid. Verify your PROSPEO_API_KEY environment variable.
RATE_LIMIT_EXCEEDED rate_limit medium Too many requests in a short period. Wait before retrying.
ACCOUNT_RESTRICTED api_client_error critical Account is suspended or restricted. Contact support@prospeo.io.
INTERNAL_ERROR api_server_error high Prospeo server error. Retry once; if it persists contact support.
NETWORK_ERROR api_server_error high Could not reach the Prospeo API (connection refused, DNS failure, etc.).
TIMEOUT api_server_error medium Request exceeded PROSPEO_TIMEOUT (default 30 s). Retry or increase the timeout.

Credit Usage

Credits are deducted per successful API call. Failed lookups (person not found, company not found) and free_enrichment: true responses do not consume credits.

ActionCreditsNotes
enrich_person — email only 1 Default behaviour
enrich_person — mobile (with or without email) 10 When enrich_mobile: true. Email is included at no extra cost.
enrich_company 1 Per successful enrichment
search_person 1 Per page that returns results
search_company 1 Per page that returns results
get_account_info 0 Always free
💡

Use get_account_info before a large search batch to confirm you have enough credits. A search across 10 pages costs 10 credits.
